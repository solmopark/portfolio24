  • 코딩을 시작한 계기는?

    창의적인 일에 관심이 많고 성장과 도전하는 것에 두려움을 느끼지 않는 성격입니다.
    우연히 코딩을 접했을 때 상상한 것을 웹구현을 통해 눈으로 직접 확인 가능하다는 점과 작은 디테일에 따라 변화하는 모습 등 계속해서 새로운 도전이 동반된다는 점이 저에게 매력적으로 다가왔습니다.
    웹 구현 결과물에서 응용하기 좋고 유용해 보이는 코드를 보며 구현해보면 좋겠다고 생각했습니다.

  • 어떤 업무를 했었나요?

    ① 웹진
    : (사보 완성 후) 각 페이지별 디자인 확인 ➔ 사보 디자인 맞추어 웹진 디자인 후 웹페이지 코딩 및 반응형 코딩(CSS 및 JS 추가) ➔ 내부 컨펌 ➔ 기관 컨펌
    ② 뉴스레터
    : 웹진 최종 컨펌 후 해당 메인 비주얼 반영하여 디자인 / 메인이미지 없는 경우 클립아트로 계절감 맞춰 디자인 ➔ 시안 컨펌 요청 ➔ 컨펌 후 이미지 업로드. 이미지 절대경로 사용하여 코딩 ➔ 기관 테스트 메일 발송 ➔ 타스온 최종 발송 or 예약 발송
    ③ 카드뉴스
    : 웹진 원고 전체 컨펌 후, 카드뉴스 원고 전달 받음 ➔ 작업 후 내부 컨펌 ➔ 컨텐츠팀 컨펌 ➔ 기관 송부 및 컨펌 요청 ➔ 웹진 게시
    ④ 이벤트
    : 기획안 작성 후 기관 컨펌 요청 ➔ 해당 기획안 디자인(웹게시용, sns게시용) ➔ 이벤트 폼 작성 ➔ 웹진 게시 및 sns 게시 요청
    ⑤ 결과보고서
    : 각 과업별 매월 애널리틱스를 활용해 웹 사이트 방문자 및 이용현황 결과보고

  • 앞으로 목표는 있나요?

    전 회사에선 html/css와 디자인 역량을 키우며 사용자에게 브랜드의 가치를 전달하고 가독성 등 편리하게 이용할 수 있도록 만드는 것이 중요하다는 것을 깨달았습니다.
    하지만 웹진 특성상 JAVA script 및 Jquery 등의 사용 빈도가 적을 수 밖에 없어 웹 구현을 하고 싶은 저로썬 많이 아쉬웠습니다.
    웹 구현 실력 향상을 위해 컴퓨터 언어공부와 실무경험을 통해 최상의 디지털 경험을 제공할 수 있는 웹퍼블리셔에서 프론트엔드가 되고싶습니다. 또한 여러 사람과 소통하고 고민 하며 한계를 두지 않고 성장해나가겠습니다
